Continuous Deployment - Overview

Deployment Process

Trigger: Push to main, master, or develop branch

Deployment Strategy:

  • Platform: Netlify
  • Method: Git Integration (automatic)
  • Configuration: netlify.toml

How it works:

  1. Code pushed to main branch
  2. GitHub Actions verifies build (CD workflow)
  3. Netlify detects push automatically
  4. Netlify builds and deploys to production
  5. Live site updated: 696d3132a4208000081bd3bf--keen-trifle-8c918c.netlify.app

CD: Build & Deploy

Stage 1: Build Verification (GitHub Actions)

1. Install dependencies (npm ci) 2. Build Convex backend 3. Build frontend for production 4. Verify build succeeds

Stage 2: Netlify Deployment (Automatic)

1. Netlify detects Git push 2. Runs build command: npm install && npm run build 3. Publishes dist/ directory 4. Deploys to CDN 5. Updates production site
Deployment time: ~3-4 minutes total

Error Handling & Feedback Loops

What happens when a build fails?

Pipeline Stopping

CI/CD pipeline stops at the failing stage. No further stages execute. Deployment is blocked.

Error Reporting

GitHub Actions logs show detailed errors with file paths and line numbers. Workflow summary shows which job failed.

Example failure scenario:

Stage 2: Type Check fails â†’ Error: Type 'string' is not assignable to type 'number' â†’ File: src/components/Chatbot.tsx:45 â†’ Pipeline stops â†’ Deployment blocked â†’ Developer gets notification â†’ Fix locally and push again
